# 성적 분석 기능 추천 10가지

> 작성일: 2026-02-11
> 대상: PS-GradeM (현장 모의고사 성적 분석 시스템)
> 기준: 현재 코드(Code.gs, Index.html) 분석 후 도출

---

## 1차 추천 (분석 확장 + 시스템 기능)

### 1. 회차별 성적 추이 분석 — 선택 확률 ~90% / 난이도 중

수험생 최대 관심사: "내 성적이 오르고 있는가?"

- 현재는 단일 회차만 분석 가능
- `Scores` 시트에 여러 회차 데이터가 쌓이므로, 동일 학생의 과거 시험을 모아 꺾은선 그래프로 표현
- 총점/과목별 점수 추이, 석차 변동, 백분위 변화
- **백엔드**: `getStudentExamList`로 응시 이력을 이미 가져오고 있으므로, 각 회차별 요약 데이터(총점, 석차, 상위%)만 추가 계산
- **프론트엔드**: 새 탭 "성적 추이" + Chart.js line chart

### 2. 목표 점수 시뮬레이터 — 선택 확률 ~80% / 난이도 하

합격 피라미드는 "지금 어디인지"만 보여주고, "얼마나 더 올려야 하는지"는 안 보여줌.

- 합격예상점수(`exam.cut`)와 내 과목별 점수 차이 분석
- 합격권 진입까지 필요한 총 점수 갭
- 과목별 "현실적 목표 점수" 제안 (취약 과목 우선 배분)
- 예: "형사법 +7.5점, 헌법 +5점 → 합격가능권 진입"
- **백엔드**: `Code.gs:1478`에서 `hasCutLine`과 `subjectStats`를 이미 계산 중이므로 갭 분석 로직만 추가
- **프론트엔드**: 종합현황 탭 하단 또는 별도 탭

### 3. 과목별 세부영역 취약점 분석 — 선택 확률 ~70% / 난이도 중

현재 오답 분석은 문항 단위까지만. "어떤 영역이 약한지" 패턴은 미제공.

- PRD에 `ItemAnalysis` 시트 설계에 `분류(대)`, `분류(중)` 컬럼이 있으나, 현재 `ITEM_COL`에는 6개 컬럼만 정의
- **수정**: `ITEM_COL`에 `CATEGORY_L`, `CATEGORY_M` 추가
- 영역별 정답률 집계 → "수사법 40%, 형법총론 80%" 같은 영역별 강약점 히트맵
- **시트 스키마 확장 필요**

### 4. 성적 분석 리포트 저장 (PDF/이미지) — 선택 확률 ~60% / 난이도 하~중

현재 수험표 인쇄(`printAdmitTicket`)는 있지만, 성적 분석 결과 전체를 저장/공유하는 기능 없음.

- 학원생이 부모님이나 스터디원에게 결과 공유 시 필요
- **방법 1** (간단): `window.print()` + `@media print` CSS로 인쇄용 레이아웃 적용
- **방법 2** (고급): html2canvas + jsPDF CDN으로 PDF 생성

### 5. 동점자/유사 점수대 과목 구성 비교 — 선택 확률 ~50% / 난이도 하

경쟁자 탭은 총점과 문항 응답만 표시. "같은 점수대인데 왜 석차가 다른지" 과목별 비교 미제공.

- 나와 총점 ±5점 이내 경쟁자의 과목별 점수를 겹쳐 비교
- "경쟁자 A는 형사법이 높고 헌법이 낮다 / 나는 반대" 같은 전략적 인사이트
- 레이더 차트에 경쟁자 평균을 추가 dataset으로 표현
- **백엔드 수정 없음**: `competitors` 배열(`Code.gs:1496-1521`)에 `s1, s2, s3` 과목별 점수가 이미 존재하나 프론트엔드에서 총점만 노출 중

## 2차 추천 (학습 진단 + 실전 도구)

### 6. 오답 노트 자동 생성 — 선택 확률 ~85% / 난이도 하

수험생이 시험 후 가장 먼저 하는 일이 오답 정리.

- 틀린 문항만 과목별로 묶어 학습용 오답 노트 형태로 정리
- 문항번호 / 정답 / 내 답 / 정답률 / 난이도를 한눈에 표시
- 정답률 높은 순 정렬 (쉬운 문제부터 먼저 복습 유도)
- 인쇄 전용 CSS 적용하여 종이 오답 노트로 출력 가능
- **백엔드 수정 없음**: `itemAnalysis` 배열에서 `my === 'X'`인 문항만 필터

### 7. 합격권 학생 대비 과목별 Gap 분석 — 선택 확률 ~75% / 난이도 하

"전체 평균과 비교"보다 "합격하는 사람들과 비교"가 훨씬 유의미.

- `Code.gs:1212-1215`에서 이미 `top10Scores`, `top30Scores` 분리, 과목별 `top10Avg`, `top30Avg` 계산 중
- 프론트엔드에서 테이블 숫자로만 보여줄 뿐, 과목별 gap을 시각적으로 강조하지 않음
- 가로 막대 차트로 과목별 `합격권 평균 - 내 점수` gap 표시
- 양수(내가 앞섬) → 파란색, 음수(내가 부족) → 빨간색
- 가장 gap이 큰 과목을 "최우선 보강 과목"으로 강조
- **백엔드 수정 0줄**: `subjectStats`의 `top30Avg`와 `my` 차이만 시각화

### 8. 난이도별 정답률 분석 — 학습 유형 진단 — 선택 확률 ~70% / 난이도 하~중

"총 정답률 75%"보다 "쉬운 문제 95% / 어려운 문제 30%"가 학습 방향 설정에 유용.

- `ITEM_COL.DIFFICULTY`(`Code.gs:68`)에 난이도 데이터 이미 존재
- 상/중/하 그룹핑하여 난이도별 내 정답률 vs 전체 평균 정답률 비교 테이블
- 학습 유형 자동 진단:
  - **기본 실수형**: 쉬운 문제 정답률 < 전체 평균 → "기본 개념 복습 우선"
  - **고난도 도전형**: 어려운 문제 정답률 높음 → "기본기 유지 + 심화 학습"
  - **안정형**: 모든 난이도에서 평균 이상 → "현재 패턴 유지"

### 9. 문항 구간별 오답 집중도 — 시간 관리 진단 — 선택 확률 ~60% / 난이도 하

실전에서 시간 부족으로 뒷문항을 못 풀거나 찍는 패턴은 점수에 큰 영향.

- 과목별 문항을 앞(1~33%) / 중(34~66%) / 뒤(67~100%) 구간으로 나누어 오답률 비교
- 뒷 구간 오답 비율이 앞 구간의 2배 이상이면 "시간 배분 전략 조정 필요" 경고
- **추가 데이터 불필요**: `itemAnalysis`의 `num`(문항번호)과 `my`(정오) 활용

### 10. 기대 득점 vs 실제 득점 비교 — 선택 확률 ~55% / 난이도 하

문항별 전체 정답률로 "평균적인 수험생이라면 받았을 점수"(기대 득점)를 계산.

- **기대 득점** = Σ (문항별 correctRate × 배점)
- **실제 득점** = 내 실제 총점
- 비교 결과:
  - 실제 > 기대: 실력 이상 발휘 (어려운 문항에서 선전)
  - 실제 ≈ 기대: 평균적 수행
  - 실제 < 기대: 컨디션/실수 문제 (쉬운 문항에서 실점)
- 과목별 분리 시 "형사법은 기대 이상, 헌법은 기대 이하" 같은 과목별 컨디션 진단 가능
- `itemAnalysis`의 `correctRate`, `points`, `my` 필드로 계산
