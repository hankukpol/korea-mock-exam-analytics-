📄 [PRD] 모의고사 성적 분석 및 합격 예측 시스템 (MVP)1. 프로젝트 개요 (Overview)프로젝트명: PS-GradeM (Police & Fire Station Grade Management)목표: 경찰 및 소방 수험생에게 자신의 모의고사 성적, 석차, 약점 분석, 합격 가능성을 시각적으로 제공하여 학습 동기를 부여하고 학원 신뢰도를 높임.타겟 유저: 학원 수강생 (경찰직 / 소방직)핵심 기술:Frontend: HTML5, CSS3, JavaScript (Chart.js 활용 추천)Backend: Google Apps Script (doGet, doPost)Database: Google Spreadsheets2. 사용자 시나리오 (User Flow)접속: 웹사이트 링크(GAS 배포 URL) 접속로그인/인증: 이름 + 전화번호 뒷자리 or 수험번호 입력직렬/회차 선택:직렬 선택 (경찰 / 소방 / 구급 등)확인하고 싶은 모의고사 회차 선택 (예: 2026 시즌1 1회차)대시보드 확인: 종합 점수, 전체 석차, 합격 예측 피라미드 확인상세 분석: 과목별 성적, 레이더 차트, 오답률 Top 5 확인지역 분석: 내가 지원한 지역(예: 대구청, 경북청) 내에서의 내 위치 확인3. 상세 기능 요구사항 (Functional Requirements)A. 메인 대시보드 (종합 분석)합격 예측 (피라미드 차트):참고 이미지 1, 8: 전체 응시자 데이터 분포를 기반으로 내 점수가 [합격 확실 / 유력 / 가능 / 도전] 중 어디에 위치하는지 시각화.로직: 사전에 설정된 합격 컷라인 점수와 내 점수 비교.내 성적 요약 카드:총점, 평균, 백분위(상위 %), 전체 석차 (내 등수 / 전체 응시자).참고 이미지 5: 전체 평균 vs 내 점수 막대 비교.B. 직렬별 맞춤 데이터 처리경찰: 형사법, 경찰학, 헌법 (3과목) / 공채, 경채 구분소방: 소방학개론, 관계법규, 행정법 (3과목) / 공채, 구급, 구조 구분기능: 사용자가 선택한 직렬에 따라 보여주는 과목명과 만점 기준이 동적으로 변경되어야 함.C. 지역별 경쟁 분석 (Regional Rank)지역 내 석차:전체 석차가 아닌, 나와 같은 '응시 지역(예: 대구)'을 선택한 학생들 간의 석차 별도 표기.지역별 평균 비교: 해당 지역 응시생 평균 vs 내 점수 비교.D. 과목별 상세 분석 (Subject Detail)레이더 차트 (육각형 그래프):참고 이미지 4: 과목별(또는 영역별) 내 점수 vs 전체 평균 vs 상위 10% 평균을 겹쳐서 표현. 균형 잡힌 학습인지 한눈에 파악.영역별 강/약점:각 과목의 세부 영역(예: 영어-문법, 독해 / 한국사-근현대사 등)별 정답률 표시.E. 문항 분석 (Item Analysis)오답률 Top 5 (Killer Questions):참고 이미지 3, 9: 전체 응시자들의 정답률이 가장 낮은 5문제를 보여주고, 내가 맞았는지 틀렸는지(OX) 표시.성적표 리스트:참고 이미지 2: 문항 번호 | 내 답 | 정답 | 채점 결과 | 정답률 | 난이도 표시.4. 데이터베이스 구조 (Google Sheets 설계)구글 스프레드시트를 DB로 활용하므로 시트 구조가 매우 중요합니다.Sheet 1: DB_Student (학생 정보)수험번호이름전화번호직렬(경찰/소방)지원지역비밀번호(접속용)21889이동근1234경찰_일반남대구1234Sheet 2: DB_ExamMaster (시험 정보)시험ID시험명시행일과목1과목2과목3합격권점수P251225년 12월 경찰 모의고사2025-12-27형사법경찰학헌법230Sheet 3: DB_Score (성적 데이터 - UI/UX 개발 가이드 (참고 이미지 기반)부원장님께서 직접 코딩하시거나 지시하실 때 참고할 구체적인 라이브러리입니다.차트 라이브러리: Chart.js (필수)이미지에 있는 레이더 차트(육각형), 막대 그래프, 파이 차트를 구현하기에 가장 가볍고 쉽습니다. CDN으로 바로 불러와서 사용 가능합니다.피라미드 차트 구현:Chart.js로도 가능하지만, CSS로 삼각형 모양의 <div>를 층층이 쌓고, 내 위치에 해당하는 층에 active 클래스를 주어 색상을 바꾸는 방식이 더 깔끔합니다.디자인 프레임워크: Bootstrap 5 또는 Tailwind CSS모바일/PC 반응형 웹을 만들기 위해 필수입니다. 카드 형태의 UI(성적표 박스)를 만들기 쉽습니다.6. 개발 로드맵 (Step-by-Step)1단계: 데이터 및 백엔드 (GAS)구글 시트에 위 4번 항목의 더미 데이터를 입력합니다.GAS에서 doGet() 함수를 만들어 HTML 페이지를 띄웁니다.getStudentScore(studentId, examId) 함수를 만들어, 특정 학생의 성적과 전체 통계(평균, 등수)를 계산하여 JSON으로 리턴하는 API를 짭니다.2단계: 프론트엔드 - 조회 화면로그인 화면(수험번호 입력)을 만듭니다.JavaScript fetch() 또는 google.script.run을 사용하여 백엔드에서 데이터를 가져옵니다.가져온 데이터로 텍스트(점수, 석차)를 먼저 화면에 뿌려봅니다.3단계: 시각화 (Chart.js 적용)성적 분포도: 전체 학생 점수 배열을 가져와 히스토그램을 그립니다.레이더 차트: 내 과목별 점수 vs 전체 평균 점수 데이터를 매핑합니다.오답률: 문항별 정답률 데이터를 정렬하여 상위 5개를 테이블로 만듭니다.4단계: 직렬/지역 필터링 로직 추가데이터를 가져올 때 filter() 함수를 사용하여 '내 지역'과 같은 학생들만 추려내어 '지역 석차'를 다시 계산하는 로직을 추가합니다.
