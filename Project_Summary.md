# 모의고사 성적 분석 및 합격 예측 시스템 구축 프로젝트 리포트

본 문서는 시스템의 초기 설계부터 심화 기능 구현까지의 모든 과정을 기록한 최종 워크스루입니다.

---

## 1단계: 데이터베이스 설정 (Google Sheets)
사용하시는 스프레드시트에 아래 **4개의 시트**를 생성하고, **첫 번째 줄(1행)에 헤더**를 정확한 순서로 입력해 주세요. (시트 이름의 대소문자를 구분해 주세요.)

| 시트 이름 | 헤더 (1행 내용) |
| :--- | :--- |
| **Students** | 수험번호, 이름, 직렬, 지원지역, 비밀번호 |
| **Exams** | 시험ID, 시험명, 시행일, 과목1, 과목2, 과목3, 합격예상점수 |
| **Scores** | 시험ID, 수험번호, 과목1점수, 과목2점수, 과목3점수, 총점, 평균 |
| **ItemAnalysis** | 시험ID, 문항번호, 정답, 배점, 정답률, 난이도, 영역분류 |

> [!IMPORTANT]
> - 열(Column)의 순서가 바뀌면 데이터 분석 로직이 어긋날 수 있으니 주의해 주세요.

---

## 2단계: 백엔드 API 개발 (Google Apps Script)
핵심 비즈니스 로직인 `getStudentReport(examId, studentId)`를 `Code.gs`에 구현했습니다.

- **석차 산출**: 해당 시험 전체 응시자 대비 내 등수 계산
- **지역 석차**: 동일 '지원지역' 수험생들 사이의 상대적 위치 산출
- **백분위 계산**: 상위 % 지표 제공
- **합격 예측**: 마스터 시트의 합격예상점수와 비교하여 PASS/FAIL 판정
- **킬러 문항 추출**: 정답률이 가장 낮은 상위 5개 문항 자동 필터링

---

## 3단계: 프론트엔드 UI 및 시각화 (Index.html)
Bootstrap 5와 Chart.js를 활용하여 현대적인 대시보드를 구축했습니다.

- **글래스모피즘 디자인**: 투명하고 세련된 카드 레이아웃 적용
- **레이더 차트**: 내 과목별 점수와 전체 평균 점수를 다각형 그래프로 비교
- **반응형 리포트**: 모바일과 데스크톱 모두에서 최적화된 성적 리포트 확인 가능

---

## 4단계: 심화 기능 (시각화 고도화)
사용자 경험을 극대화하기 위한 시각적 지표를 추가했습니다.

- **4단계 합격 예측 피라미드**: 
  - **확실 / 유력 / 가능 / 도전** 4개 층으로 구성
  - 학생의 점수 위치에 따라 해당 층의 색상이 진하게(Highlight) 변하도록 CSS와 JS 로직을 추가
- **킬러 문항 분석 테이블**:
  - 오답률 Top 5 문항의 영역과 난이도, 정답률을 명시하여 학습 방향 제시
